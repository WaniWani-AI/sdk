The chat embed is a self-contained IIFE bundle that drops the WaniWani chat into any website. It talks directly to `app.waniwani.ai`, so no backend proxy is required on your side.

<Note>
**Platform feature.** Requires a WaniWani project token (`wwp_...`). [Get one at app.waniwani.ai](https://app.waniwani.ai) or read more about the [Platform](/platform/overview).
</Note>

The embed mounts **inline** wherever you place a `<div data-waniwani-embed></div>` marker. You control the size by sizing that container with CSS. There is no floating bubble.

## Quick start

Two pieces: a mount point and the script tag.

```html
<!-- Where the chat appears, sized by your CSS -->
<div data-waniwani-embed style="height: 600px; max-width: 480px;"></div>

<!-- Load the embed. Auto-initializes when `data-token` is set. -->
<script
  src="https://cdn.waniwani.ai/embed.js"
  data-token="wwp_..."
  data-title="Assistant"
  data-welcome-message="How can I help?"
  data-suggestions="Get started,Pricing,Talk to sales"
  defer
></script>
```

That's it. The chat mounts inside your marker div via Shadow DOM (so its styles never leak into your page).

## Programmatic init

Drop the `data-token` attribute and call `init()` manually if you need to compute config at runtime:

```html
<div data-waniwani-embed></div>

<script src="https://cdn.waniwani.ai/embed.js" defer></script>
<script>
  window.addEventListener("DOMContentLoaded", () => {
    window.WaniWani.chat.init({
      token: "wwp_...",
      title: "Assistant",
      welcomeMessage: "How can I help?",
      suggestions: ["Get started", "Pricing", "Talk to sales"],
    });
  });
</script>
```

`init()` returns an instance with `destroy()` and `sendMessage(text)`. You can also call `window.WaniWani.chat.destroy()` and `window.WaniWani.chat.sendMessage("...")` directly on the global.

## Self-hosting the embed

If you'd rather not load from the WaniWani CDN, pin a specific SDK version and serve the file yourself:

```bash
bun add @waniwani/sdk
cp node_modules/@waniwani/sdk/dist/chat/embed.js public/waniwani-embed.js
```

Then reference your local copy:

```html
<div data-waniwani-embed></div>
<script src="/waniwani-embed.js" data-token="wwp_..." defer></script>
```

## Configuration

Every option is settable two ways: as a `data-*` attribute on the script tag, or as a key in the `init()` argument. Programmatic values override `data-*` attributes when both are present.

| Option | `data-*` attribute | Type | Notes |
|---|---|---|---|
| `token` | `data-token` | string | Required. Public project token (`wwp_...`). |
| `api` | `data-api` | string | Override chat API URL. Defaults to `https://app.waniwani.ai/api/mcp/chat`. |
| `mcpServerUrl` | `data-mcp-server-url` | string | Override the MCP server endpoint. |
| `title` | `data-title` | string | Header title. Defaults to `"Assistant"`. |
| `welcomeMessage` | `data-welcome-message` | string | Greeting shown before the first message. |
| `placeholder` | `data-placeholder` | string | Input field placeholder. |
| `suggestions` | `data-suggestions` | string[] (comma-separated in attr) | Initial suggestion chips. |
| `enableThreadHistory` | `data-enable-thread-history` | boolean | Persist threads in IndexedDB. Defaults to `false`. |
| `css` | `data-css` | string | URL to a custom stylesheet loaded into the shadow root. |
| `theme.primaryColor` | `data-primary-color` | string | Brand color (bubble, send button). |
| `theme.backgroundColor` | `data-background-color` | string | Panel background. |
| `theme.textColor` | `data-text-color` | string | Default text color. |
| `theme.fontFamily` | `data-font-family` | string | Custom font stack. |

Full type definitions: [`EmbedConfig`](https://github.com/WaniWani-AI/sdk/blob/main/src/chat/web/embed/config.ts) and [`ChatTheme`](https://github.com/WaniWani-AI/sdk/blob/main/src/chat/web/%40types.ts).

## Sizing

The embed fills its container. Give the `<div data-waniwani-embed>` a definite size via CSS:

```html
<!-- Fixed height -->
<div data-waniwani-embed style="height: 600px;"></div>

<!-- Bounded by max-height -->
<div data-waniwani-embed style="max-height: 80vh;"></div>

<!-- Flex column child -->
<div style="display: flex; flex-direction: column; max-height: 800px;">
  <div data-waniwani-embed style="flex: 1 1 auto; min-height: 0;"></div>
</div>
```

If the parent is fully unbounded the chat grows with content. A `ResizeObserver` mirrors the container's `max-height` onto the embed so padding and borders are respected.

## How it works

1. The script reads `data-token` (and other `data-*` overrides) off itself.
2. On `DOMContentLoaded` (or immediately, if the DOM is already ready), it locates `[data-waniwani-embed]` and mounts the chat inside it via Shadow DOM.
3. The chat calls `app.waniwani.ai/api/mcp/chat` with the project token and streams responses back through Server-Sent Events.
4. Conversation history lives only in memory by default; set `enableThreadHistory` to persist threads in IndexedDB on the user's device.

No server-side integration on your side. If you want React-component-level control instead of the script tag, see [Chat in React](/chat/react).
