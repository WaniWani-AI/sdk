For React apps where you want full control over layout, theming, and lifecycle, import the chat directly from `@waniwani/sdk/chat`.

<Note>
**Platform feature.** Requires a WaniWani project token (`wwp_...`). [Get one at app.waniwani.ai](https://app.waniwani.ai) or read more about the [Platform](/platform/overview).
</Note>

## Install

```bash
bun add @waniwani/sdk
```

Add the stylesheet to your app once (e.g. in `_app.tsx` or your root layout):

```ts
import "@waniwani/sdk/chat/styles.css";
```

## Layouts

Three layout primitives ship from `@waniwani/sdk/chat`. Pick the one that fits your page:

```tsx
import { ChatBar, ChatCard, ChatEmbed } from "@waniwani/sdk/chat";
```

| Component | What it renders | Good for |
|---|---|---|
| `ChatBar` | A compact bar pinned at the bottom of its container that expands upward into a chat panel when clicked. | Marketing pages, app footers. |
| `ChatCard` | An always-visible card with a header. Sized via `width` / `height`. | Sidebars, dashboards, dedicated chat panes. |
| `ChatEmbed` | A borderless chat that fills its parent container. Bring your own backend `api`. | Embedding inside existing layouts that already provide chrome. |

There is no floating-bubble layout. If you want a fixed-corner widget, wrap one of the above in your own positioned container.

## ChatBar

```tsx
import { ChatBar } from "@waniwani/sdk/chat";

export default function Page() {
  return (
    <ChatBar
      apiKey="wwp_..."
      title="Assistant"
      welcomeMessage="How can I help?"
      suggestions={["Get started", "Pricing", "Talk to sales"]}
    />
  );
}
```

`ChatBar` talks to the WaniWani hosted backend by default. Override `api` if you proxy through your own server.

## ChatCard

```tsx
import { ChatCard } from "@waniwani/sdk/chat";

<ChatCard
  apiKey="wwp_..."
  title="Sales"
  width={420}
  height={600}
/>;
```

## ChatEmbed

Bring-your-own-backend variant. `api` is required; there is no default.

```tsx
import { ChatEmbed } from "@waniwani/sdk/chat";

<ChatEmbed
  api="/api/my-chat"
  body={{ environmentId, sessionId }}
  theme={{ backgroundColor: "#fff" }}
/>;
```

## Theming

Themes are objects keyed by role. Merge with the default theme via `mergeTheme`:

```tsx
import { ChatBar, mergeTheme, DEFAULT_THEME } from "@waniwani/sdk/chat";

const theme = mergeTheme(DEFAULT_THEME, {
  primaryColor: "#03d916",
  backgroundColor: "#fffaf0",
  textColor: "#1a1a1a",
});

<ChatBar apiKey="wwp_..." theme={theme} />;
```

Full theme keys: see [`ChatTheme`](https://github.com/WaniWani-AI/sdk/blob/main/src/chat/web/%40types.ts).

## How it works

`ChatBar` and `ChatCard` post to the WaniWani hosted chat endpoint (`app.waniwani.ai/api/mcp/chat`) using your project token, and stream responses back through Server-Sent Events. Pass an `api` prop to proxy through your own backend.

`ChatEmbed` is backend-agnostic: pass any `api` endpoint that implements the [Vercel AI SDK chat protocol](https://ai-sdk.dev) and the component handles the rest.
