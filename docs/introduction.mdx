`@waniwani/sdk` plugs into an MCP server and gives you three things:

- **Event tracking.** Automatic `tool.called` events (name, duration, status, input, output, client info) plus custom events, batched and forwarded to the WaniWani dashboard.
- **Flows.** A typed state machine compiled into a single MCP tool. Pause on interrupts, branch on conditions, persist state across turns — all keyed by the MCP session.
- **Knowledge base.** Ingest markdown files, search with embeddings. One function call from any tool handler.

## Two ways to start

| Path | When to use |
| --- | --- |
| **Managed project** | WaniWani provisions a repo, deploys to Vercel, and auto-tunnels local dev to ChatGPT/Claude. Start from the template. |
| **External project** | You already have an MCP server and want to add tracking, flows, or knowledge base to it. |

Both paths use the same SDK. The difference is infrastructure.

## SDK entry points

```
@waniwani/sdk       # Core client (tracking, config, KB)
@waniwani/sdk/mcp   # Server framework (createFlow, withWaniwani, registerTools)
@waniwani/sdk/kb    # Knowledge base client (ingest, search)
```
