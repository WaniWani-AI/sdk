<Warning>
`createTool`, `createResource`, and `registerTools` are **legacy APIs**. They are still exported from `@waniwani/sdk/mcp` for back-compat with existing customer MCPs but are no longer documented for new code. They will move to `@waniwani/sdk/legacy` in a future minor release.

**For new code, use [`createFlow`](/flows/overview) instead.** A flow compiles into a single MCP tool that handles multi-step conversations, widget rendering via `showWidget`, and state persistence — without the need to hand-register separate tools and resources.
</Warning>

## What these APIs did

- **`createResource`** declared a UI resource (an HTML template hosted somewhere) that an MCP host could render as a widget inside chat (ChatGPT, Claude Desktop, etc.).
- **`createTool`** declared a tool with structured input/output, optionally bound to a `createResource` so the model could trigger a widget render.
- **`registerTools`** bulk-registered an array of tools onto an MCP server.

The widget-rendering model — tool returns `_meta.openai/outputTemplate` or `_meta.ui/resourceUri` → host iframes the HTML — is the path we built on for the original product. It is now superseded by flows.

## Migration to `createFlow`

The typical legacy pattern:

```ts
// Legacy
const pricingUI = createResource({ id: "pricing", baseUrl: "https://app.com", htmlPath: "/widgets/pricing" });
const showPricing = createTool({
  resource: pricingUI,
  description: "Show pricing comparison",
  inputSchema: { postalCode: z.string() },
}, async ({ postalCode }) => ({ text: "Pricing loaded", data: { postalCode, plans } }));
await registerTools(server, [pricingUI, showPricing]);
```

Equivalent flow:

```ts
// New
import { createFlow, MemoryKvStore, START, END } from "@waniwani/sdk/mcp";

const pricingFlow = createFlow({
  id: "pricing_quote",
  title: "Pricing Quote",
  description: "Show pricing for a postal code",
  state: { postalCode: z.string().describe("Postal code") },
})
  .addNode("ask", () => interrupt({ postalCode: { question: "What's your postal code?" } }))
  .addNode("show", (state) => showWidget(pricingUI, {
    data: { postalCode: state.postalCode!, plans },
    description: "Pricing comparison",
  }))
  .addEdge(START, "ask")
  .addEdge("ask", "show")
  .addEdge("show", END)
  .compile({ store: new MemoryKvStore() });

await pricingFlow.register(server);
```

Flow benefits: state persistence, branching, server-side validation, single tool surface for the model, funnel analytics in the free tier.

## If you're stuck on the legacy path

These APIs are not going away in this release. They are marked `@deprecated` in JSDoc (your IDE will surface that) and grouped here in the docs so we can clearly point new users at `createFlow`. Existing apps using `createTool`/`createResource` will keep working without any code changes.

When the legacy entry point split happens, the import path will change from `@waniwani/sdk/mcp` to `@waniwani/sdk/legacy`, and you'll get a clear migration window.
