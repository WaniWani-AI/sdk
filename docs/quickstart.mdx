The same `createFlow` code runs in three modes. Start with mode 1 (OSS, no API key) and upgrade later — nothing changes except where state lives.

## Mode 1: Open source (no API key)

<Steps>
  <Step title="Install the SDK">
    ```bash
    bun add @waniwani/sdk
    ```
  </Step>

  <Step title="Write a flow">
    ```ts flow.ts
    import { createFlow, END, MemoryKvStore, START } from "@waniwani/sdk/mcp";
    import { z } from "zod";

    export const onboardingFlow = createFlow({
      id: "onboarding",
      title: "User Onboarding",
      description: "Use when a new user wants to get started.",
      state: {
        email: z.string().describe("Work email"),
        useCase: z.string().describe("What they want to build"),
      },
    })
      .addNode("ask_email", () =>
        interrupt({ email: { question: "What's your work email?" } }),
      )
      .addNode("ask_use_case", () =>
        interrupt({
          useCase: {
            question: "What do you want to build?",
            suggestions: ["Analytics", "Support", "Lead capture"],
          },
        }),
      )
      .addEdge(START, "ask_email")
      .addEdge("ask_email", "ask_use_case")
      .addEdge("ask_use_case", END)
      .compile({ store: new MemoryKvStore() });
    ```
  </Step>

  <Step title="Register with your MCP server">
    ```ts
    import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
    import { onboardingFlow } from "./flow";

    const server = new McpServer({ name: "my-mcp", version: "0.0.1" });
    await onboardingFlow.register(server);
    ```
  </Step>
</Steps>

State lives in memory and resets on restart. Fine for development and tests. For production self-hosting, see [Mode 3](#mode-3-self-hosted-production). For hosted state with zero infrastructure, see [Mode 2](#mode-2-free-tier-with-api-key).

## Mode 2: Free tier (with API key)

<Steps>
  <Step title="Get a free API key">
    1. Open [app.waniwani.ai/mcp](https://app.waniwani.ai/mcp) and sign in.
    2. Create a project and copy the `wwk_...` key (shown only once).
    3. Add to your environment:

    ```bash .env
    WANIWANI_API_KEY=wwk_...
    ```
  </Step>

  <Step title="Drop the store argument">
    ```diff
    - .compile({ store: new MemoryKvStore() });
    + .compile();
    ```

    With `WANIWANI_API_KEY` set, `createFlow` automatically uses `WaniwaniKvStore` — flow state persists on `app.waniwani.ai`, survives restarts, and shows up in your dashboard.
  </Step>

  <Step title="(Optional) Auto-track every tool">
    One call wraps your server so every `tools/call` produces a `tool.called` event with duration, status, input/output, and session correlation.

    ```ts
    import { withWaniwani } from "@waniwani/sdk/mcp";

    withWaniwani(server);
    ```

    Without an API key, `withWaniwani` still wraps tools but tracking is a silent no-op.
  </Step>

  <Step title="Verify">
    Trigger any tool call, then open the [dashboard](https://app.waniwani.ai). Events appear within seconds.
  </Step>
</Steps>

## Mode 3: Self-hosted production

Use OSS only with a production-grade `KvStore` adapter — Redis, Upstash, Cloudflare KV, DynamoDB, anything with `get`/`set`/`delete`.

```ts
import { createFlow } from "@waniwani/sdk/mcp";
import { Redis } from "@upstash/redis";

const redis = new Redis({ url: process.env.UPSTASH_URL!, token: process.env.UPSTASH_TOKEN! });

const upstashStore = {
  async get(key: string) { return redis.get(key); },
  async set(key: string, value: unknown) { await redis.set(key, value); },
  async delete(key: string) { await redis.del(key); },
};

const flow = createFlow({ /* … */ }).compile({ store: upstashStore });
```

See [KV store adapters](/flows/kv-store) for ready-made recipes (Redis, Upstash, Cloudflare KV, DynamoDB).

## Starter template

A ready-to-go project with the SDK pre-wired, including a sample flow and dev tunnel for ChatGPT/Claude:

```bash
git clone https://github.com/WaniWani-AI/mcp-distribution-template.git my-mcp-server
cd my-mcp-server
bun install
bun dev
```

The template ships in free-tier mode by default. To run OSS-only, replace the `WaniwaniKvStore` import with `MemoryKvStore` (or your own adapter) and drop the `WANIWANI_API_KEY` env var.
