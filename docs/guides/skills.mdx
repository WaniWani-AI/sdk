## WaniWani SDK

### Install

<CodeGroup>
  ```bash npx
  npx skills add WaniWani-AI/sdk -s waniwani-sdk
  ```

  ```bash bunx
  bunx skills add WaniWani-AI/sdk -s waniwani-sdk
  ```
</CodeGroup>

The skill is installed into your project's `.claude/skills/` directory (or equivalent for other agents).

### What's included

The skill teaches your agent how to:

- **Set up the SDK** -- install, configure env vars, create a client singleton
- **Track events** -- `withWaniwani` auto-tracking, custom events, sessions
- **Build flows** -- multi-step conversational state machines with interrupts, validation, conditional edges
- **Create tools with widgets** -- `createResource`, `createTool`, React hooks for widget UIs
- **Add a knowledge base** -- ingest markdown, search with embeddings, build FAQ tools
- **Embed chat** -- React component and script-tag embed with theming

### Usage

After installing, just describe what you want in natural language. Your agent will use the skill automatically:

- *"Add WaniWani tracking to my MCP server"*
- *"Create a multi-step onboarding flow"*
- *"Add a knowledge base FAQ tool"*
- *"Embed a chat widget on my landing page"*

## OpenAI App Submission

When submitting your MCP server to the ChatGPT App Store, you need to provide Tool Justification and Test Cases documents. The WaniWani dashboard handles this for you: go to your project's submission page and use the built-in Codex assistant to generate a `chatgpt-app-submission.json` file, then upload it to auto-fill the form.

The generated file includes:

1. **Tool Justification** -- annotations (Read Only, Open World, Destructive) with justifications for every tool
2. **Test Cases** -- positive and negative test scenarios for OpenAI reviewers

## Claude App Submission

When submitting your MCP server to the [Claude Connectors Directory](https://claude.com/docs/connectors/building/submission), the `claude-submission` skill drives the whole process: it audits your repo into a copy-paste cheat sheet, verifies your widgets emit the correct `ui.domain` for Claude, checks tool descriptions against Claude's review criteria, and walks the 6-page submission form. A companion `local-mcp-claude` skill registers your dev server in Claude Desktop so you can test locally first.

<CodeGroup>
  ```bash npx
  npx skills add WaniWani-AI/sdk -s claude-submission -s local-mcp-claude
  ```

  ```bash bunx
  bunx skills add WaniWani-AI/sdk -s claude-submission -s local-mcp-claude
  ```
</CodeGroup>

The skill captures hard-won gotchas, including:

- **The `ui.domain` render check** -- Claude validates widget HTML against `sha256(<server-url>/mcp).claudemcpcontent.com`. Apps built on the skybridge template already emit this correctly; the skill shows you how to **verify it with the Claude user-agent** (a plain curl returns your deploy host and looks broken when it isn't).
- **Review-criteria audit** -- the read/write tool split, and behavioral tool-description language that gets a tool rejected.
- **The Google Form traps** -- the account-revert reset, checkbox minimum-N requirements, favicon verification, and which media fields take Drive links vs uploads.

See the [Claude submission deployment guide](/deployment/claude-submission) for the full walkthrough.
