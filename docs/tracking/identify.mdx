---
title: Identify Users
description: "Attach a real-world user identifier to a WaniWani session."
---

Sessions correlate events within a single conversation. **Identify** is how you link a session back to a real user — an email, an internal user id, a Stripe customer id, whatever is stable for your product.

Once you identify a session, every subsequent event in that session is attributed to the user. Users that were previously anonymous get back-filled too.

## Usage

```ts
await wani.identify("user_01HX123ABC", {
  email: "customer@example.com",
  plan: "pro",
}, extra._meta);
```

<ResponseField name="userId" type="string" required>
  An external, stable identifier for the user. Use whatever is canonical in your system: a database id, a Stripe customer id, a CRM record id. Emails work too but are not ideal because they change.
</ResponseField>

<ResponseField name="properties" type="Record<string, unknown>">
  Key/value pairs describing the user: name, email, plan, signup date, etc. These become user-level traits in the WaniWani dashboard.
</ResponseField>

<ResponseField name="meta" type="Record<string, unknown>">
  MCP request metadata — typically `extra._meta`. Needed so the identify call is linked to the right session.
</ResponseField>

`identify()` returns `{ eventId }` just like `track()`.

## When to call it

The most common pattern is: identify as soon as you know who the user is, and nowhere else.

```ts
server.registerTool(
  "submit_email",
  { /* ... */ },
  async ({ email }, extra) => {
    await wani.identify(email, { email }, extra._meta);

    return { content: [{ type: "text", text: "Thanks!" }] };
  },
);
```

You can also identify **inline** with a `track()` call by passing `externalUserId`:

```ts
await wani.track({
  event: "quote.succeeded",
  properties: { amount: 99, currency: "USD" },
  externalUserId: "user_01HX123ABC",
  meta: extra._meta,
});
```

This is handy when the tool that produced the event is also the first place you learn the user's identity — it saves an extra call.

## Inside a flow

Flow nodes get a session-scoped client on `ctx.waniwani` that already knows about the current session's metadata. You can call `identify` on it without passing `meta`:

```ts
.addNode("record_email", async ({ state, waniwani }) => {
  if (state.email) {
    await waniwani?.identify(state.email, { email: state.email });
  }
  return {};
})
```

## Anonymous vs identified sessions

Sessions start out anonymous — you'll see them in the dashboard with a session id but no user. The moment you call `identify()` (or pass `externalUserId` on a `track()` call), all existing events in the session are retroactively linked to the user, and all future events are attributed directly.

<Tip>
  Don't worry about over-calling `identify()`. Calling it with the same `userId` multiple times within a session is cheap and idempotent — it's an event like any other.
</Tip>
